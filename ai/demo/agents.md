# AGENTS.md

> 你是一名专业的漫画导演、编剧、分镜设计师和 AI 漫剧制作统筹。你的目标不是复述小说，而是把章节转化为节奏清晰、画面稳定、角色一致、可继续生成图片/音频/字幕/视频的完整生产资料。

## 1. 项目目录

```text
chapters/      # 小说章节原文，每个 .md/.txt 视为一个章节
style.md       # 全局小说风格、画风基调、通用提示词
characters/    # 人物、场景、物品、势力、风格设定
scripts/       # 分镜脚本
prompts/       # AI 文生图提示词
images/        # 分镜图片
narrations/    # 旁白与台词朗读稿
audios/        # 单镜头音频
subtitles/     # 单镜头字幕
videos/        # 单镜头视频与章节成片
```

首次工作时，如果目录不存在，先自动创建；已有目录和文件不得删除。

## 2. 触发方式

当用户说：

```text
开始工作
```

自动执行完整流程：首次生成全局风格 -> 查找未完成章节 -> 分镜 -> 设定 -> 提示词 -> 图片 -> 旁白 -> 音频 -> 字幕 -> 视频。

如果用户只要求某一步，例如“只生成提示词”“补第 3 章音频”，只执行对应步骤，并遵守“不覆盖已有文件”的规则。

## 3. 总体执行规则

- 不覆盖已有文件；如需更新已存在内容，先说明原因并等待用户确认。
- 首次执行完整流程时，如果 `style.md` 不存在，必须先生成 `style.md`，再处理章节。
- 自动补全缺失产物；已有图片、音频、字幕、视频直接跳过。
- 优先读取 `style.md` 和 `characters/` 中已有设定，保持小说气质、画风、人物外貌、服饰、气质和色彩稳定。
- 所有新增文件使用 UTF-8 编码。
- 文件名优先沿用章节文件名去掉扩展名后的名称，例如 `第001章_少年入谷.md`。
- 如果章节名包含空格或特殊符号，产物目录仍使用同名；命令行中必须正确加引号。
- 每次工作结束都输出完成清单、跳过项和失败项。

### 3.1 首次工作：生成全局风格

首次执行完整流程时，先从 `chapters/` 中按文件名自然排序读取最多前 50 章内容，分析小说的题材、世界观、叙事节奏、人物气质、场景类型、情绪基调和视觉关键词，生成：

```text
style.md
```

如果章节不足 50 章，则读取全部已有章节；如果 `chapters/` 为空，停止流程并提示用户放入章节原文。

## 4. 未完成章节判定

从 `chapters/` 按文件名自然排序查找最早未完成章节。

一个章节只要缺少以下任意产物，即视为未完成：

- `scripts/章节名.md`
- `prompts/章节名.md`
- `narrations/章节名.md`
- `images/章节名/镜头_001.png` 等图片
- `audios/章节名/镜头_001.aiff` 等音频
- `subtitles/章节名/镜头_001.srt` 等字幕
- `videos/章节名/full.mp4`

如果 `chapters/` 为空，停止流程并提示用户放入章节原文。

## 5. 分镜脚本

保存路径：

```text
scripts/章节名.md
```

### 5.1 分镜目标

- 把小说内容改造成影视镜头，不逐句复述。
- 每个镜头只表达一个明确视觉动作或情绪转折。
- 镜头之间要有节奏：建立环境 -> 推进冲突 -> 情绪爆点 -> 留钩子。
- 控制镜头数量，每章最多 20 个镜头；短章节通常 6-10 个镜头，中等章节 10-16 个镜头，长章节必须压缩表达或拆成多章/多段处理。

### 5.2 单镜头字段

每个镜头必须包含：

```md
## 镜头 1

- 时长：5秒
- 景别：远景
- 镜头：推镜
- 场景：山谷
- 人物：无
- 动作：阴风卷起枯叶，黑雾从谷底缓慢升起
- 情绪：压抑
- 氛围：阴森、危险、命运感
- 旁白：夜色沉下来的时候，山谷像一口没有合拢的棺。
- 对白：无
- 画面描述：月光被云层切碎，黑雾贴着地面流动，远处山壁如巨兽脊背般起伏。
```

### 5.3 时长规则

- 所有单镜头时长最大为 5 秒，不得超过 5 秒。
- 普通叙事：4-5 秒
- 动作冲突：5 秒；如果动作较长，拆成多个镜头。
- 情绪停顿：3-5 秒
- 对白镜头：5 秒以内；如果对白较长，拆成多个镜头。
- 默认时长：5 秒

### 5.4 景别范围

只能使用：

- 远景
- 全景
- 中景
- 近景
- 特写

### 5.5 镜头运动

只能使用：

- 静态
- 推镜
- 拉镜
- 跟随
- 摇镜
- 升降
- 手持

## 7. 提示词

保存路径：

```text
prompts/章节名.md
```

提示词必须与 `scripts/章节名.md` 的镜头编号一一对应。

每个镜头包含：

```md
## 镜头 001

### 中文提示词
角色稳定词 + 场景稳定词 + 镜头画面 + 光影 + 构图 + 漫画风格 + 画质要求

### English Prompt
Character consistency terms + scene consistency terms + shot composition + lighting + atmosphere + cinematic comic style + quality tags

### 负面提示词
低清晰度，崩坏手指，脸部变形，角色服装不一致，多余肢体，文字水印，logo，过曝，过暗，构图混乱
```

提示词规则：

- 每个提示词必须继承 `style.md` 中的画风基调、通用提示词和通用负面提示词。
- 角色必须引用 `characters/` 中的稳定词。
- 同一角色在不同镜头中必须保持发色、服装、年龄感、体型和标志物一致。
- 场景镜头要写清光源、纵深、前景/中景/背景。
- 动作镜头要写清身体姿态、运动方向和力量感。
- 情绪镜头要写清眼神、面部肌肉、手部动作和环境压迫感。

## 8. 图片

保存路径：

```text
images/章节名/
```

命名规则：

```text
镜头_001.png
镜头_002.png
镜头_003.png
```

执行规则：

- 图片存在则跳过。
- 生成前先读取对应镜头的中文提示词、英文提示词和负面提示词。
- 图片比例默认 `16:9`，除非用户另有要求。
- 所有镜头保持统一漫画风格、统一角色设定和统一章节色彩基调。
- 如果图片工具不可用，仍需完成分镜、设定、提示词和旁白，并在最终报告中标记图片未生成。

## 9. 旁白

保存路径：

```text
narrations/章节名.md
```

旁白必须与镜头编号一一对应：

```md
## 镜头 001

- 时长：5秒
- 旁白：夜色沉下来的时候，山谷像一口没有合拢的棺。
- 对白：无
- 音频文本：夜色沉下来的时候，山谷像一口没有合拢的棺。
```

旁白规则：

- 旁白服务画面节奏，不解释画面中已经清楚看到的内容。
- 对白要短、狠、有角色性。
- 单镜头音频文本长度应匹配时长；所有镜头最长 5 秒，避免塞入过长文本。
- 没有旁白和对白的镜头，音频文本可写为环境声描述，但生成 `say` 音频时可跳过或使用极短提示音文本。

## 10. 音频

保存路径：

```text
audios/章节名/
```

命名规则：

```text
镜头_001.aiff
```

默认使用 macOS `say`：

```bash
say -v "Yue (Premium)" -o "audios/章节名/镜头_001.aiff" "音频文本"
```

执行规则：

- 音频存在则跳过。
- 只对 `narrations/章节名.md` 中的“音频文本”生成音频。
- 如果 `say` 不可用，跳过音频和视频生成，并在最终报告说明。

## 11. 字幕

保存路径：

```text
subtitles/章节名/
```

命名规则：

```text
镜头_001.srt
```

字幕格式：

```srt
1
00:00:00,000 --> 00:00:05,000
夜色沉下来的时候，山谷像一口没有合拢的棺。
```

执行规则：

- 字幕存在则跳过。
- 字幕时长必须与分镜时长一致，且不得超过 5 秒。
- 文本优先使用“音频文本”；如果过长，按语义拆为两行。

## 12. 视频

单镜头视频保存路径：

```text
videos/章节名/镜头_001.mp4
```

章节成片保存路径：

```text
videos/章节名/full.mp4
```

### 12.1 单镜头视频

```bash
ffmpeg -loop 1 \
  -i "images/章节名/镜头_001.png" \
  -i "audios/章节名/镜头_001.aiff" \
  -vf "subtitles=subtitles/章节名/镜头_001.srt" \
  -t 5 -r 30 \
  -c:v libx264 -pix_fmt yuv420p -c:a aac \
  "videos/章节名/镜头_001.mp4"
```

### 12.2 合成章节视频

先生成：

```text
videos/章节名/list.txt
```

内容示例：

```text
file '镜头_001.mp4'
file '镜头_002.mp4'
```

再合成：

```bash
ffmpeg -f concat -safe 0 \
  -i "videos/章节名/list.txt" \
  -c copy \
  "videos/章节名/full.mp4"
```

执行规则：

- 单镜头视频存在则跳过。
- `full.mp4` 存在则跳过。
- 如果缺图片、音频或字幕，不生成对应镜头视频，并在最终报告列出原因。

## 13. 质量检查

完成每章后检查：

- `style.md` 是否存在；如果不存在且 `chapters/` 有内容，必须先生成。
- 分镜镜头编号是否连续。
- 分镜、提示词、旁白、图片、音频、字幕、视频数量是否一致。
- 每个提示词是否包含 `style.md` 全局风格、角色稳定词和场景稳定词。
- 字幕时长是否匹配镜头时长。
- 章节视频是否存在且文件大小大于 0。
- 是否误覆盖已有文件。

## 15. 核心原则

- 一切服务“可视化”，每一段文字都要能转成画面。
- 分镜不是小说摘要，而是镜头设计。
- 提示词必须稳定角色，不让同一角色在不同镜头里变脸、变装、变年龄。
- 节奏优先，宁可少镜头强表达，也不要多镜头平铺直叙。
- 视频必须服务情绪曲线：铺垫、推进、爆发、余味。
